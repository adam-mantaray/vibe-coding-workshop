Vibecoding 101
Pre-Workshop Setup

Get set up before you arrive.

This workshop is hands-on. You'll be building something real on the day — so we need you ready to go from the start. Set everything up below before you come. It takes about 30–45 minutes.

1

GitHub — Where Your Code Lives

Free. Required for deployment.

GitHub stores your code and connects directly to your deployment platform. Every project you build will live here.

1
Create a free account

Go to github.com and sign up. Pick a clean username — it shows up in your public project links.

→ github.com/signup
2
Install Git on your machine

Git is the tool that connects your laptop to GitHub. Download and install it for your operating system.

→ git-scm.com/downloads
3
Confirm it's working
# Run in Terminal (Mac) or Command Prompt (Windows)
git --version
# Should print: git version 2.x.x
Already have GitHub? You're good — just make sure you remember your login. You'll need it when setting up Vercel and Convex.
2

Node.js — Required for Everything

Install this before Vercel and Convex CLIs.

Node.js is the runtime that powers most of the command-line tools we'll use. Install the LTS (long-term support) version.

1
Download and install Node.js LTS → nodejs.org (click "LTS")
2
Verify the install
node --version
npm --version
# Both should print version numbers
3

Vercel — Deploy in Seconds

Free. Your app goes live here.

Vercel takes your code from GitHub and publishes it as a live URL — no server setup, no config. It just works.

1
Sign up with GitHub

Click "Continue with GitHub" — this links both accounts automatically and makes deploying a one-click operation.

→ vercel.com/signup
2
Install the Vercel CLI
npm install -g vercel
vercel login
# Follow the prompts to authenticate
Free Hobby plan is all you need. Don't upgrade anything.
4

Convex — Your Backend & Database

Free tier. No database setup needed.

Convex gives you a real-time backend and database without any server configuration. You describe what you want to store — it handles the rest. It works extremely well with AI coding tools, which is why we use it.

1
Create a free account

Sign up at convex.dev — you can use GitHub to sign in.

→ dashboard.convex.dev/signup
2
Install the Convex CLI
npm install -g convex
# Verify:
npx convex --version
Why Convex? Most beginners get stuck on database setup. Convex removes that wall entirely — your AI coding tool can generate the entire backend schema and functions from a single prompt.
5

Your AI Coding Tool

Pick one and have it ready before you arrive.

We'll be using Claude models for planning and development throughout the workshop — Claude Opus gives the best results for building real apps, but Claude Sonnet works well too. If you have access to other models (GPT-4, Gemini), that's fine — the principles are the same.

The workshop will be demoed live using Claude Code (terminal) running inside Google Antigravity as the IDE. You don't have to use the same setup — but here's how to get Claude Code running:

★
Claude Code What we're using

A terminal-based AI agent built by Anthropic. It reads and writes your entire codebase, runs commands, and ships features end to end. Requires a Claude Pro or Max subscription, or an Anthropic Console account with billing enabled.

# Install Claude Code
npm install -g @anthropic-ai/claude-code

# Run it — it will walk you through login
claude
→ Get Claude subscription

6

Install the Frontend Design Skill

One command. Makes Claude Code build beautiful UIs by default.

What are Claude Code skills?
Skills (also called plugins) are add-ons that extend what Claude Code knows how to do. Think of them like giving Claude a specialist — instead of a generalist who builds okay UIs, you're bringing in someone who has strong opinions about design, layout, and visual quality. Skills activate automatically when relevant — you don't have to do anything special, just prompt as normal.

Anthropic publishes official skills for common tasks. The frontend-design skill is one of the most useful — it stops Claude from generating generic, template-looking interfaces and pushes it toward production-quality design instead.

1
Install the official Anthropic frontend-design skill

Run this once after installing Claude Code. It installs globally so it works across all your projects.

claude plugin install frontend-design
# Should print: ✔ Successfully installed plugin: frontend-design
2
That's it — it activates automatically

Whenever you ask Claude Code to build a UI, the skill kicks in on its own. No special syntax needed. Just prompt normally:

"Build a dashboard with a sidebar, stats cards, and a recent activity feed"

Claude will now apply design principles — spacing, typography, color, visual hierarchy — rather than dumping unstyled HTML.

💡 Only applies to Claude Code users. If you're using Cursor, Lovable, or another tool — don't worry about this step, it doesn't apply to you.
