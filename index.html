Step 1 — 5 to 15 min

Start with the spec.

The number one reason vibe coding projects fall apart is jumping straight into building. A spec takes 10 minutes and saves hours of fixes.

Your PRD answers 3 questions
  • What is this product?
  • Who is going to use it?
  • What will they actually do on it?
Then break it into phases. Phase 1 is core functionality only. Everything else is Phase 2 or later.
Use this prompt now
"I want to build [your idea].
Ask me questions to help write
a product spec with phases
and screens."
Let it ask. Answer honestly. The output becomes your build guide.
Step 2 — 25 to 35 min

See it before you build it.

Pick a design style first. Generate HTML mockups in 2–3 variations. Changing a mockup takes seconds — changing real code takes hours.

Why HTML mockups? Faster than Figma to iterate. Lower variation when you move to the real build — you can reuse the code directly. And the AI is very good at generating them.
Step 3 — 35 to 50 min

Build it properly.

Two rules that eliminate most beginner mistakes before they happen.

Stack
Next.js Convex Convex Auth Vercel Claude Code
Tell Claude your stack at the start of every session. Every single time — it matters.
Two rules — no exceptions
1
Build backend functions in parallel with the frontend. Never at the end — you will regret it.
X
No mock data. Ever. Hard-coded data will haunt every future feature you try to build.
Step 4 — 50 to 60 min

Deploy first. Then fix.

First pass is always around 75%. That is fine. Ship it immediately, get a live URL, then iterate on the real thing — not localhost.

$ vercel --prod live in ~30 seconds
Vague — slow to fix
"The timer is broken"
Specific — fast to fix
"The timer resets when I click pause. It should stop at the current time and resume from there when I click play again."
The more specific your bug prompt, the faster the fix. Describe the exact behavior — not just that something is wrong.
