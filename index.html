Things AI Will
Confidently Get Wrong

And you won't catch unless you know what to look for

🔓

Security? Nah.

API keys in frontend code. No input validation. Auth that looks right but isn't.

🎭

Fake packages

"npm install super-auth-helper" — that package doesn't exist. The AI made it up. It does this a lot.

📦

Kubernetes for your todo app

AI loves over-engineering. Microservices, Docker Compose, Redis caching — for 3 users. Just... no.

🍝

Spaghetti architecture

It writes code that works RIGHT NOW. Not code that works when you add the next feature.

True story

I once let Cursor write an entire auth system for a client project. Looked perfect. Worked in dev. Shipped it.

Turned out any user could see any other user's data by changing one number in the URL.

Knowledge would have caught that in 10 seconds. — the kind of mistake you only make once

How I Actually Prompt

Not a framework. Just how I think before I type.

my actual process
1. What's the data?
   "Users have projects. Projects have tasks. Tasks have status + assignee."

2. What's the flow?
   "Sign up → create project → add tasks → drag to done."

3. What do I NOT need yet?
   "No teams. No billing. No notifications. Just tasks."

4. What tools?
   "Next.js 15, Convex, Tailwind, shadcn/ui. Nothing else."

The AI fills in the rest. But it needs YOU to set the boundaries.
