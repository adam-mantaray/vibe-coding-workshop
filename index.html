Step 1 — 5 to 15 min

Start with the spec.

The #1 reason vibe coding projects fall apart: jumping straight into building. A spec takes 10 minutes and saves hours of fixes.

Your PRD answers 3 questions

  • What is this product?
  • Who is going to use it?
  • What will they do on it?
Then break it into phases — Phase 1 is core only. Everything else is Phase 2+.

Prompt to use right now

"I want to build [your idea]. Ask me questions to help write a product spec with phases and screens."
Let it ask. Answer honestly. The output becomes your build guide.
Step 2 — 25 to 35 min

See it before you build it.

Pick a design style first. Generate HTML mockups in 2–3 variations. Iterate visually — changing a mockup takes seconds, changing real code takes hours.

Step 3 — 35 to 50 min

Build it properly.

Two rules that will save you from most beginner mistakes.

Stack

Next.js Convex Convex Auth Vercel Claude Code
Tell Claude your stack at the start of every session. Every time.

Two rules — no exceptions

⚡ Build backend functions in parallel with the frontend. Never at the end.
🚫 No mock data. Ever. If it's hard-coded, it will haunt you.
Step 4 — 50 to 60 min

Deploy first. Then fix.

First pass is always ~75%. That's fine. Ship it, get a live link, then iterate on the real thing — not localhost.

$ vercel --prod → live in 30 seconds
❌ Vague bug report
"The timer is broken"
✅ Specific bug report
"The timer resets when I click pause. It should stop at the current time and resume from there when I click again."
The more specific your bug prompt, the faster the fix. Treat it like a ticket, not a complaint.
